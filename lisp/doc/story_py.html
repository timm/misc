A Tour of ezr.py

From one lonely value to explainable multi-objective optimization, in five hundred lines of Python. Same arc as the Lisp tour; different notation; identical numbers.

Once upon a time there was a value. It did not know where it was among the other values. Then it found it was part of a row, and there were many rows, and there were patterns around: patterns in the other values at the same place in other rows. Slopes. Hills. Groups. Before long, the value knew where it stood.
Where we end up

2. Reading one row from a file

Text on disk needs typed values in memory. The parser in ezr.py is two functions.

thing: coerce one cell

def thing(txt):
  txt = txt.strip()
  b = lambda s: {"true": 1, "false": 0}.get(s.lower(), s)
  for f in [int, float, b]:
    try: return f(txt)
    except ValueError: pass
Strip, then try int, float, bool table. Fall through returns the string unchanged.
>>> thing("304")
304
>>> thing("18.5")
18.5
>>> thing("?")
'?'
>>> thing("red")
'red'
>>> thing("true")
1
"?" is just a string, not a sentinel type. Python's missing marker piggybacks on the string fall-through: every numeric parse fails, the bool table doesn't contain it, so "?" comes back as a string. The Lisp port uses the symbol ?; Python uses the string. Both work because neither matches a number.
thing("") returns '', not None and not an error. Empty cells survive as empty strings. Callers that care (like csv below) must filter.

csv: a generator with comments and blanks

def csv(f, clean=lambda s: s.partition("#")[0].split(",")):
  with open(f, encoding="utf-8") as file:
    for s in file:
      r = clean(s)
      if any(x.strip() for x in r): yield [thing(x) for x in r]
Three moves in five lines.
  • Comments. s.partition("#")[0] drops from the first # to end-of-line. "1,2,3 # skip" → "1,2,3 ".
  • Blank lines. any(x.strip() for x in r) skips rows where every cell is empty (including comment-only lines).
  • Typing. Each surviving cell through thing.

4. Names decide roles

The header encodes the schema. Two rules.

  • First character. Uppercase → Num; lowercase → Sym.
  • Last character. +/- goal; ! class label; X skip; else feature.

Num and Sym

class Num:
  def __init__(i, txt="", a=0):
    i.txt, i.at, i.n = txt, a, 0
    i.mu, i.m2, i.sd, i.heaven = 0, 0, 0, txt[-1:] != "-"

class Sym:
  def __init__(i, txt="", a=0):
    i.txt, i.at, i.n, i.has = txt, a, 0, {}

def Col(txt="", a=0):
  return (Num if txt[0].isupper() else Sym)(txt, a)
i as self-name is an ezr.py convention (inherited from the Lisp port). Python's usual self would work identically. txt[-1:] (slice, not index) is safe on empty string — returns "" rather than raising.
The suffix controls heaven at construction. Check both directions.
>>> Num("Mpg+").heaven
True
>>> Num("Lbs-").heaven
False
heaven is a bool, not 0/1 as in the Lisp port. int(True) == 1 so arithmetic works, but the repr differs.

Cols: partition the header

class Cols:
  def __init__(i, names):
    i.names, i.all = names, [Col(txt, j) for j, txt in enumerate(names)]
    i.klass = next((c for c in i.all if c.txt[-1] == "!"), None)
    i.xs    = [c for c in i.all if c.txt[-1] not in "+-!X"]
    i.ys    = [c for c in i.all if c.txt[-1] in "+-!"]

5. A column that remembers

Can a column report its mean without keeping every value? Yes. Welford, 1962.

Welford, in math

  μₙ  = μₙ₋₁ + (v − μₙ₋₁) / n
  M₂ₙ = M₂ₙ₋₁ + (v − μₙ₋₁) · (v − μₙ)
  σ   = √( M₂ / (n − 1) )

Two increments per sample — no list of prior values needed. σ is recoverable from M₂ and n on demand.

Welford, in code

def summarize(c, v, w=1):
  if v == "?": return v
  if Sym == type(c): c.has[v] = w + c.has.get(v, 0)
  elif w < 0 and c.n <= 2: c.n = c.mu = c.m2 = c.sd = 0
  else:
    c.n += w
    delta = v - c.mu
    c.mu += w * delta / c.n
    c.m2 += w * delta * (v - c.mu)
    c.sd = (max(0, c.m2) / (c.n - 1))**.5 if c.n > 1 else 0
mu updates before m2 reads it — Welford's identity uses the NEW mean for the m2 term. Not a bug.

trace: five adds, two subs

Undo two adds with weight −1.

Landed exactly where we were at step 3. Ch 12 uses this to shuffle rows between sets without recomputing.

summarize(Num(), "?") returns "?" without touching state. Missing values propagate, never aggregate.

Sym: a dict instead of μ and M₂

>>> s = Sym()
>>> for v in ('red','red','blue'): summarize(s, v)
>>> s.has
{'red': 2, 'blue': 1}

summarize vs add

Two related verbs you'll see through the rest of the tour:

  • summarize(c, v, w) updates one column's summary.
  • add(it, v, w=1) dispatches on type. For a Data it invalidates the centroid cache, updates every column, and appends the row. For a Col it delegates to summarize.
  • sub(it, v) is add(it, v, -1). Symmetric remove.

6. Mid and spread

def mid(c):
  return c.mu if Num==type(c) else max(c.has, key=c.has.get)

def spread(c):
  if Num==type(c): return c.sd
  n = sum(c.has.values())
  return -sum(v/n * log2(v/n) for v in c.has.values())
Num: mid is mean, spread is SD. Sym: mid is mode, spread is Shannon entropy.
  H = − Σᵢ  pᵢ · log₂ pᵢ

>>> mid(n), spread(n)
(20.0, 10.0)
>>> mid(s), spread(s)
('red', 0.9182958340544896)

By hand: √(200/2) = 10.0. Entropy for 2-red / 1-blue = 0.918 bits. Matches.

Entropy in bits because log2. Max entropy for 2 categories is 1.0 bit (50/50 split). We're at 0.918 because 2/3 vs 1/3 is less mixed.
mid(Num()) on an empty column returns 0 (the default mu), not an error. Callers that care must check n > 0 first.

7. Where do I stand?

  z    = (v − μ) / σ
  norm = 1 / (1 + exp( −1.7 · clamp(z, −3, 3) ))

def norm(c, v):
  if v == "?": return v
  z = (v - c.mu) / (c.sd+1e-32)
  z = max(-3, min(3, z))
  return 1/(1 + exp(-1.7*z))
z-score, clamped to ±3 (99.7% of a Gaussian), sigmoided at scale 1.7 (approximates the Gaussian CDF). Output is "fraction of column ≤ v".

9. Distance to heaven

  d = ( Σᵢ |xᵢ|^p  /  n ) ^ (1/p)           Minkowski
  disty(row) = d( |norm(c, r[c.at]) − c.heaven|  for c in ys )

def minkowski(items):
  tot, n = 0, 1e-32
  for item in items: tot, n = tot + item**the.p, n + 1
  return (tot/n) ** (1/the.p)

def disty(d, r):
  return minkowski(abs(norm(c, r[c.at]) - c.heaven) for c in d.cols.ys)
For each goal: |norm − heaven|. Roll up by Minkowski at exponent the.p (default 2: Euclidean).

11. Distance between rows

def distx(d, r1, r2):
  return minkowski(aha(c, r1[c.at], r2[c.at]) for c in d.cols.xs)

def aha(c, u, v):
  if u == v == "?": return 1
  if Sym == type(c): return u != v
  u, v = norm(c, u), norm(c, v)
  u = u if u != "?" else (0 if v > 0.5 else 1)
  v = v if v != "?" else (0 if u > 0.5 else 1)
  return abs(u - v)
Syms: same-or-different. Nums: normalized absolute difference.
Missing values are pessimistic. When one value is known and the other is missing, the unknown is pushed to the extreme furthest from the known. Pessimism penalizes unknowns without throwing them away.

12. Labels cost money

In real problems y-values are expensive. Build the car, drive it. Ship the release, wait. Features are cheap; labels are not. So we pretend y is hidden and the learner decides which row to pay for next.

Four pieces: seed, decide, prune, loop.

seed: warm_start

def warm_start(d, rows, label):
  lab = clone(d, rows[:the.learn.start])
  lab.rows.sort(key=lambda r: disty(lab, label(d, r)))
  n = int(sqrt(len(lab.rows)))
  return lab, clone(d, lab.rows[:n]), clone(d, lab.rows[n:])
  1. Pay for the first 4 labels (the.learn.start=4). Build a Data from them; μ, σ, etc. now summarize four points.
  2. Sort those 4 by disty. Best first.
  3. Top √4 = 2 rows are "elite", rest are "ordinary".
  4. Return three Datas: all labels, best, rest.
label is a callback. In offline testing it is lambda _,r: r — the row already carries its y-columns. In an online setting it would call a human, run an experiment, or query a simulator.

decide: acquireWithCentroid

def acquireWithCentroid(d, best, rest, r):
  return distx(d, r, mids(best)) - distx(d, r, mids(rest))
One comparison, one sign.
  • negative: r is closer to best's centroid → label it.
  • positive: r is closer to rest's centroid → skip it.
  • zero: tie.

mids(best) is the centroid: one mid per column, packed into a row-shaped list, memoized on _centroid. add/sub invalidate by setting _centroid = None. Comparisons after the first cost a dict lookup per column.

prune: rebalance

def rebalance(best, rest, lab):
  if len(best.rows) > sqrt(len(lab.rows)):
    best.rows.sort(key=lambda r: disty(lab, r))
    add(rest, sub(best, best.rows[-1]))

Best grows every time acquireWithCentroid fires. Cap it at √(|lab|): if over, sort by disty, pop the worst, push into rest. sub and add are the Welford-symmetric mutation pair.

Why √N and not log N or fixed? At N labels, √N is small enough to stay discriminating, big enough that one noisy row does not poison the centroid. Empirically, √N lands right for budgets in the 30-100 range.

loop: active

def active(d, score=acquireWithBayes, label=lambda _,r:r):
  rows = d.rows[:]
  shuffle(rows)
  lab, best, rest = warm_start(d, rows, label)
  pool = rows[the.learn.start:][:the.few]
  cap = the.learn.start + the.learn.budget
  for r in pool:
    if len(lab.rows) >= cap: break
    if score(lab, best, rest, r) < 0:
      add(lab, add(best, label(d, r)))
      rebalance(best, rest, lab)
  lab.rows.sort(key=lambda r: disty(lab, r))
  return lab
  1. Shuffle: defeat file-order bias.
  2. Seed 4 with warm_start.
  3. Pool = remaining rows capped at the.few=128.
  4. For each: if score < 0, label, add to best, rebalance. Stop at cap = start + budget (4 + 50 = 54).
  5. Final sort by disty so lab.rows[0] is the best-known.
score is a strategy slot. Default is acquireWithBayes (Naive Bayes likelihood ratio). acquireWithCentroid is the distance version we just dissected.

13. Hills on the landscape

32 labels; enough to draw a map.

tree-cuts: what cuts a column offers

def treeCuts(c, rs):
  vs = [r[c.at] for r in rs if r[c.at] != "?"]
  if not vs: return []
  return set(vs) if Sym == type(c) else [sorted(vs)[len(vs)//2]]

Sym: all unique values. Num: the median only.

Median-only is not laziness; it is regularization. At 32 labels, trying every threshold would overfit. One cut per numeric column keeps the tree honest.

Score = n_left · σ_left + n_right · σ_right. Lower = cleaner split. A pure-left node has σ_left = 0 on one side. treeGrow picks the column+cut with the lowest score.

14. Route a new row

A tree trained on 32 labeled rows is asked about a row it never saw. This is where the story either pays off or collapses.

def treeLeaf(t, r):
  if not t.left: return t
  v = r[t.col.at]
  go = (v != "?" and v <= t.cut) if Num == type(t.col) else (v != "?" and v == t.cut)
  return treeLeaf(t.left if go else t.right, r)
Missing values always go right (the v != "?" guard fails, which flips go to False). The Lisp port routes missing values left. Cross-port divergence; pick one convention and document it.

route best and worst rows

>>> best_r = rs[-1]                    # Mpg=50, light
>>> mid(treeLeaf(tr, best_r).ynum), treeLeaf(tr, best_r).ynum.n
(0.3635, 5)
>>> worst_r = [8, 455, 225, 73, 1, 4951, 11, 10]
>>> mid(treeLeaf(tr, worst_r).ynum), treeLeaf(tr, worst_r).ynum.n
(0.772, 6)

Best row routes to a good leaf (mean disty 0.36); worst row to a bad leaf (mean 0.77). A leaf is a mean over 5-6 rows so exact disty does not come out — but the neighborhood is correct. That is what generalization looks like.

To build a holdout protocol on top of this: split the data half-and-half, run active on train, grow a tree, rank test rows by leaf mean, reveal the top the.learn.check (default 5), return the best of those. Same contract as the Lisp validate.

16. Where we arrived

We started with the integer 42. It did not know where it stood.

We gave it friends: a row. We gave the row siblings: many rows. The header named the columns. Each column learned its middle and width in constant space via Welford. We placed every value inside its column's bell with a sigmoid-of-z. We defined heaven. We collapsed three goals into one scalar through Minkowski. We sorted. We measured rows by feature-space neighborhoods. We labeled cheaply with an acquisition function and a √N elite pile. We grew a median-regularized decision tree. We routed fresh rows through it.
