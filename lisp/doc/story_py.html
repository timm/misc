A Tour of ezr.py

From one lonely value to explainable multi-objective optimization, in five hundred lines of Python. A companion to the Lisp tour. Same arc; different notation; identical numbers.

Once upon a time there was a value. It did not know where it was among the other values. Then it found it was part of a row, and there were many rows, and there were patterns around: patterns in the other values at the same place in other rows. Slopes. Hills. Groups. Before long, the value knew where it stood.

This is a tour of ezr.py. Each stop is a few lines. Every output was captured live.

2. Reading one row from a file

Text on disk needs typed values in memory. The whole parser in ezr.py is two functions. First, thing coerces one cell:

def thing(txt):
  txt = txt.strip()
  b = lambda s: {"true": 1, "false": 0}.get(s.lower(), s)
  for f in [int, float, b]:
    try: return f(txt)
    except ValueError: pass

Strip whitespace. Try int, then float, then the bool table. Fall through returns the stripped string unchanged. Missing values stay as the literal "?".

>>> thing("304")
304
>>> thing("18.5")
18.5
>>> thing("?")
'?'
>>> thing("red")
'red'
>>> thing("true")
1

Now csv wraps thing in a generator that also handles comments and blank lines:

def csv(f, clean=lambda s: s.partition("#")[0].split(",")):
  with open(f, encoding="utf-8") as file:
    for s in file:
      r = clean(s)
      if any(x.strip() for x in r): yield [thing(x) for x in r]

Three moves in five lines:

  • Comments. s.partition("#")[0] drops everything from the first # to end-of-line, so "1,2,3 # skip me" becomes "1,2,3 ".
  • Blank lines. if any(x.strip() for x in r) skips rows where every cell is empty, including lines that were comment-only.
  • Typing. Each surviving cell goes through thing.

4. Names decide roles

The header encodes the schema. First character: uppercase → Num, lowercase → Sym. Last character: +/- goal, ! class, X skip, else feature.

def Col(txt="", a=0):
  return (Num if txt[0].isupper() else Sym)(txt, a)

class Num:
  def __init__(i, txt="", a=0):
    i.txt, i.at, i.n = txt, a, 0
    i.mu, i.m2, i.sd, i.heaven = 0, 0, 0, txt[-1:] != "-"

Suffix controls heaven at construction. One flag per goal, decided once.

5. A column that remembers

Can a column report its mean without keeping every value? Yes. Welford, 1962.

def summarize(c, v, w=1):
  if v == "?": return v
  if Sym == type(c): c.has[v] = w + c.has.get(v, 0)
  elif w < 0 and c.n <= 2: c.n = c.mu = c.m2 = c.sd = 0
  else:
    c.n += w
    delta = v - c.mu
    c.mu += w * delta / c.n
    c.m2 += w * delta * (v - c.mu)
    c.sd = (max(0, c.m2) / (c.n - 1))**.5 if c.n > 1 else 0

Three increments per sample. mu updates before m2 reads it — Welford's identity uses the new mean. The weight w makes the update symmetric: pass -1 to unwind.

>>> n = Num()

...     summarize(n, v)
...     print(f"after {v}: n={n.n} mu={n.mu} m2={n.m2}")

after 20: n=2 mu=15.0 m2=50.0
after 30: n=3 mu=20.0 m2=200.0
after 40: n=4 mu=25.0 m2=500.0

6. Mid and spread

def mid(c):
  return c.mu if Num==type(c) else max(c.has, key=c.has.get)

def spread(c):
  if Num==type(c): return c.sd
  n = sum(c.has.values())
  return -sum(v/n * log2(v/n) for v in c.has.values())

Num: mid is mean, spread is SD. Sym: mid is mode, spread is Shannon entropy.

>>> mid(n), spread(n)
(20.0, 10.0)
>>> mid(s), spread(s)
('red', 0.9182958340544896)

Check by hand: sqrt(200/2) = 10.0. Entropy for 2-red/1-blue = 0.918 bits. Matches.

7. Where do I stand?

def norm(c, v):
  if v == "?": return v
  z = (v - c.mu) / (c.sd+1e-32)
  z = max(-3, min(3, z))
  return 1/(1 + exp(-1.7*z))

z-score, clamped to ±3 (covers 99.7% of a Gaussian), sigmoided at scale 1.7 (approximates the Gaussian CDF). Output is "fraction of column ≤ v".

9. Distance to heaven

def minkowski(items):
  tot, n = 0, 1e-32
  for item in items: tot, n = tot + item**the.p, n + 1
  return (tot/n) ** (1/the.p)

def disty(d, r):
  return minkowski(abs(norm(c, r[c.at]) - c.heaven) for c in d.cols.ys)

For each goal: delta = |norm - heaven|. Roll up by Minkowski with exponent the.p (default 2). Worked on the first data row [8, 304, 193, 70, 1, 4732, 18.5, 10]:

11. Distance between rows

def distx(d, r1, r2):
  return minkowski(aha(c, r1[c.at], r2[c.at]) for c in d.cols.xs)

def aha(c, u, v):
  if u == v == "?": return 1
  if Sym == type(c): return u != v
  u, v = norm(c, u), norm(c, v)
  u = u if u != "?" else (0 if v > 0.5 else 1)
  v = v if v != "?" else (0 if u > 0.5 else 1)
  return abs(u - v)

Syms: same-or-different. Nums: normalized absolute difference. A missing value is pessimistic — assume it sits at whichever extreme is furthest from its partner.

>>> distx(d, r0, rs[-1])                      # opposite

12. Labels cost money

In real problems, y-values are expensive. You want Mpg? Build the car, drive it. Features are cheap; labels are not. So we pretend y is hidden and uncover it one row at a time. The learner decides which row to pay for next.

Four pieces: warm_start, acquireWithCentroid, rebalance, and the active loop that ties them together.

warm_start: seed four, split by disty

def warm_start(d, rows, label):
  lab = clone(d, rows[:the.learn.start])
  lab.rows.sort(key=lambda r: disty(lab, label(d, r)))
  n = int(sqrt(len(lab.rows)))
  return lab, clone(d, lab.rows[:n]), clone(d, lab.rows[n:])

Line by line:

  1. lab = clone(d, rows[:the.learn.start]): take the first 4 rows (the.learn.start=4) and build a Data with just those rows. These are our four seed labels. clone keeps the column schema and summarizes the seeds into it — we now have mu, sd, etc. on four points.
  2. lab.rows.sort(key=lambda r: disty(lab, r)): order the four by distance to heaven. Best first, worst last.
  3. n = int(sqrt(len(lab.rows))): for four rows, n=2. The top n become our "elite" pile.
  4. Return three Data objects: all labels (lab), the top two (best), the bottom two (rest).

The label callback decides how we get a y-value. In offline testing (our case) it is lambda _,r: r — the row already carries its y-columns. In an online setting it would call a human, run an experiment, or query a simulator.

acquireWithCentroid: one comparison, one sign

def acquireWithCentroid(d, best, rest, r):
  return distx(d, r, mids(best)) - distx(d, r, mids(rest))

mids(best) is best's centroid — one mid per column, packed into a row-shaped list. Same for rest. The function returns:

  • negative when r is closer to best's centroid than to rest's — a sign that r might be great;
  • positive when closer to rest — probably not worth labeling;
  • zero on a tie.

The loop treats < 0 as "label this one". Centroids of best and rest are memoized on _centroid inside each Data; add and sub invalidate them by setting _centroid to None. So the comparison costs a dict lookup per column after the first call.

rebalance: keep the elite pile small

def rebalance(best, rest, lab):
  if len(best.rows) > sqrt(len(lab.rows)):
    best.rows.sort(key=lambda r: disty(lab, r))
    add(rest, sub(best, best.rows[-1]))

best gets one new member every time the acquisition function fires. Without a cap, it would grow to include the whole labeled set and stop discriminating. So after each add, we compare |best| to sqrt(|lab|). When it's over, we:

  1. Sort best.rows by disty (best first, worst last).
  2. best.rows[-1] is the worst one in the elite pile. sub(best, ...) removes it, running the Welford update with weight -1 so the centroid corrects itself in constant time.
  3. add(rest, ...) puts the evicted row into rest, updating that centroid too.

Net effect: best stays around sqrt(N) rows of the very best labels we've paid for. The centroid of best sharpens as the labeled set grows.

active: the driving loop

def active(d, score=acquireWithBayes, label=lambda _,r:r):
  rows = d.rows[:]
  shuffle(rows)
  lab, best, rest = warm_start(d, rows, label)
  pool = rows[the.learn.start:][:the.few]
  cap = the.learn.start + the.learn.budget
  for r in pool:
    if len(lab.rows) >= cap: break
    if score(lab, best, rest, r) < 0:
      add(lab, add(best, label(d, r)))
      rebalance(best, rest, lab)
  lab.rows.sort(key=lambda r: disty(lab, r))
  return lab

Step by step:

  1. rows = d.rows[:]; shuffle(rows): copy and randomize so the seed rows and pool order aren't biased by file order.
  2. warm_start: pay for 4 seed labels; set up best and rest from them.
  3. pool = rows[the.learn.start:][:the.few]: the candidate set. Skip the 4 already labeled, cap the rest at the.few=128. The cap bounds per-iteration work and the total pool scan.
  4. cap = start + budget: absolute label ceiling (default 4 + 50 = 54).
  5. For each pool row, ask score(...). If negative:
    • add(best, label(d, r)): label r and put it in best. This nested add returns r after the summary update.
    • add(lab, ...): also add it to the full labeled set.
    • rebalance(...): evict best's worst if over quota.
  6. After the loop, sort the labeled rows by disty so callers see lab.rows[0] as the best-known row.

The score parameter is a strategy slot. Default is acquireWithBayes (Naive Bayes likelihood ratio of "best" vs "rest"). Pass acquireWithCentroid to get the distance version we just dissected.

Now run it:

>>> lab = active(d, score=acquireWithCentroid)
>>> len(lab.rows)

>>> for r in lab.rows[:3]:
...     print(f"  {disty(lab,r):.3f}  {r}")

32 labels bought three rows under disty 0.27. The learner never peeked at rows outside the labeled set. A one-twelfth budget found the top region.

Default acquisition is Bayes, not centroid
active defaults to acquireWithBayes, which builds a Naive Bayes model over labeled rows and asks "more likely to be best than rest?" The centroid version is the closest match to the Lisp port. Both land near the same budget.

13. Hills on the landscape

32 labels; enough to draw a map.

def treeCuts(c, rs):
  vs = [r[c.at] for r in rs if r[c.at] != "?"]
  if not vs: return []
  return set(vs) if Sym == type(c) else [sorted(vs)[len(vs)//2]]

Sym: all unique values. Num: median only — regularization at small N.

14. Route a new row

def treeLeaf(t, r):
  if not t.left: return t
  v = r[t.col.at]
  go = (v != "?" and v <= t.cut) if Num == type(t.col) else (v != "?" and v == t.cut)
  return treeLeaf(t.left if go else t.right, r)
>>> leaf = treeLeaf(tr, rs[-1])
>>> mid(leaf.ynum), leaf.ynum.n
(0.3635088303117473, 5)

The tree routes our best car into a leaf with mean disty 0.36 — the right neighborhood, though a leaf is a mean over 5 rows so no single row's exact disty comes out.

Use this to rank unseen rows (holdout / test set). Pay to peek at the top the.learn.check (default 5), return the best of those five. Same contract as the Lisp validate.

16. Where we arrived

We started with 42. It didn't know where it stood.

We gave it friends: a row. We gave the row siblings: many rows. The header named the columns. Each column learned its middle and width in constant space. We placed every value inside its column's bell. We defined heaven. We collapsed three goals into one scalar. We sorted. We measured rows by feature-space neighborhoods. We labeled cheaply. We grew a tree. We routed fresh rows through it.

    value  ->  row  ->  column  ->  mid/spread  ->  norm
                                                     |
                     distx <- heaven <- disty <-----+
                      |                  |
                    active -----> tree -----> treeLeaf

Every arrow is a function. Every function is visible. The lonely value walks all the way to explainable multi-objective optimization without ever leaving ezr.py.

Appendix: sections of ezr.py not in this tour

Bayes (section 3)
Naive Bayes likelihood on best vs rest is the default active-learning acquisition function. Skipped for brevity; swap acquireWithBayes for acquireWithCentroid to see it in action.
Clustering (section 6)
kmeans, kpp, half, rhalf: projection-based recursive halving as an alternative to labeled active learning. Uses distx; pays no labels.
1+1 optimization (section 8)
sa (simulated annealing), ls (local search with restart). Same scoring by disty; different search strategy. A sandbox for comparing the learner above to classical optimization.
Stats (section 0)
same (Cliff's Delta + Kolmogorov-Smirnov), bestRanks. For honest comparison of algorithm rankings.
