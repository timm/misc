### nb

`nb` is the driver. It reads rows one at a time and does two things
with each: classify it (using everything seen so far), then train
on it. Training and testing are interleaved — there is no
train/test split, no holdout, no cross-validation. That sounds
sloppy and it works surprisingly well; we will return to why in
the exercises.

```lua
local function nb(items,   all,klasses,n,nk,klass,train,seen,classify)
  klasses, n, nk = {}, 0, 0
  function klass(row) return row[all.cols.y[1].at] end
  function train(row) klasses[klass(row)]:add(row) end
  function seen(k)
    if not klasses[k] then
      nk=nk+1; klasses[k]=clone(all); klasses[k].txt=k end end
  function classify(row)
    return l.most(klasses,function(_,d)return d:likes(row,n,nk)end)end

  for row in iter(items) do
    if not all then all=Data("all",{row}) else
      seen(klass(row))
      if n > the.wait then print(classify(row), klass(row)) end
      n=n+1; train(row) end end end
```

Four inner helpers explain themselves:

- `klass(row)` — read the row's class label out of the first `y`
  column.
- `train(row)` — find the per-class summary for that label, add
  the row to it. (Uses [`clone`](#clone) the first time a class is
  seen.)
- `seen(k)` — make sure a per-class summary exists for class `k`;
  count distinct classes seen so far in `nk`.
- `classify(row)` — for each known class, ask
  [`DATA.likes`](#datalikes); return the class with the largest
  log-likelihood. `l.most` is the "argmax" helper from
  [`lib.lua`](#lib).

The outer loop is the whole story. The first row is the header —
build an all-rows `data` from it. Every subsequent row: register
its class (so a per-class summary exists), classify it if we have
seen enough rows, then train on it.

`the.wait = 5` means we skip classifying the first five rows; the
classifier needs *some* evidence before it has an opinion. We
print every guess to stdout. Counting matches gives accuracy.

```
[?]> -- one line per row, after the burn-in:
[?]> lua nb.lua --nb data/diabetes.csv | head -5
tested_positive	tested_positive
tested_positive	tested_negative
tested_positive	tested_positive
tested_negative	tested_negative
tested_positive	tested_positive
```

Two columns per line: guess, then truth. On the full file the
classifier gets 559 out of 762 right (~73%).
